# Faculty academic-validation checklist

A technically public-ready service still requires human ownership of its academic claims. Use this checklist for every faculty release and every annual handbook update.

| Faculty | Catalogue build reproducible | Static references resolve | Rule-owner review | Ambiguities documented | Public sign-off |
|---|---|---|---|---|---|
| Humanities | Yes | Partial: inherited unverified scopes remain labelled | Pending | Required | Pending |
| Engineering & the Built Environment | Yes | Partial: inherited unverified scopes remain labelled | Pending | Required | Pending |
| Law | Yes | Yes | Pending | Required | Pending |
| Science | Yes | Yes | Pending | Required | Pending |
| Health Sciences | Yes | Yes | Pending | Required | Pending |
| Commerce | Yes | Yes | Pending | Required | Pending |

For each route, the reviewer should confirm qualification credits, compulsory courses, prerequisites/co-requisites, progression thresholds, repeat/credit rules, duration and readmission treatment, elective pools, major rules, awards/distinction rules, cohort-specific differences, and every condition that requires human confirmation.

A route must remain `provisional` or `unverified` until the relevant rule owner signs it off. Passing software tests is not a substitute for institutional authority.
